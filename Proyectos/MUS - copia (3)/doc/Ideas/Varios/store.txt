/store

- Incluir datos registro.
- Validar datos:
	- Longitudes.
	- Email lógico.
	- Email no exista para otro usuario.
	- Alias no exista para otro usuario.

- Buscar email. Si ya existe para otro usuario error y return.
- buscar usuario con alias.
- Si existe y verificado es True: Usuario ya registrado con ese Alias.

- Si no existe: 
	- Generar código activación.
	- Insertarlo con verificado = false y código activación.
	- Enviar email con código activación.
	- Enviar mensaje revisar email con código.
	- Render template.

- Si existe y verificado es false: 
	- Validar que se ha introducido código en pantalla.
	- Comparar código introducido con código registro BBDD.
	- Si coincide:
		- Actualizar verificado = true.
		- Dar acceso a la aplicación.
	- Si no coincide:
		- Mensaje código de activación erróneo.
		- Render template.

/login

- Leer usuario con alias.
- Si no existe error.
- Si existe:
	Si verificado = false:
		- Error: debe introducir el código de verificación en el registro.
	Si verificado = true:
		- Dar acceso a la aplicación.
